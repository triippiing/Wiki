Mental Model The licensing chain

RHEL licensing is a chain — each link must be in place for the next to function. If dnf doesn't return repos at the end, licensing is effectively broken regardless of what intermediate status fields say.

Register→Attach→Enable repos→Patchable

Read it left-to-right when troubleshooting. Find the first link that fails — that's the one to fix. Everything downstream will recover once it's repaired.

Phase 1 Register, attach, enable repos
01 Register the system REGISTER ▶
Username / password — interactive only subscription-manager register --username=<user> --password=<pass> Activation key — enterprise standard subscription-manager register --org=<org_id> --activationkey=<key>

Use activation keys for any automated build pipeline. They scope cleanly to a subscription pool, don't expose user credentials in process lists or logs, and survive password rotations. User/pass registration is fine for ad-hoc work but not for golden-image bake-outs or kickstart files.

02 Attach a subscription ATTACH ▶
# Auto-attach picks the best-matching subscription from available pools subscription-manager attach --auto # If entitlements are stale or you've recently changed pools subscription-manager refresh

Activation keys usually attach a subscription pool automatically at registration time, so this step is often not needed. Run it only if list --consumed shows nothing attached.

03 Enable required repos ENABLE ▶
# BaseOS + AppStream — the minimum for a patchable RHEL 8/9 system subscription-manager repos \ --enable=rhel-8-for-x86_64-baseos-rpms \ --enable=rhel-8-for-x86_64-appstream-rpms # Adjust the version (8 / 9) and arch (x86_64 / aarch64 / ppc64le) as needed

RHEL 9 uses rhel-9-for-x86_64-baseos-rpms / rhel-9-for-x86_64-appstream-rpms. List all available repos for the attached subscription with subscription-manager repos --list if you're unsure of the exact name.

Phase 2 Quick audit
04 Run the full audit AUDIT ▶
# Run all five — each answers a different question about the chain subscription-manager identity # Is it registered? subscription-manager status # Is it Current / Invalid / Unknown? subscription-manager list --consumed # What's actually attached? subscription-manager repos --list-enabled # Are the right repos turned on? dnf repolist # Does dnf actually see them?

Run all five, in this order. Each line is a checkpoint along the chain. The first one that fails is your starting point for repair — don't bother fixing later steps until the earlier ones come back clean.

05 Interpret the output READ ▶
Command OK signal Failure signal
identity Shows system UUID + org "This system is not yet registered"
status Current Invalid / Unknown
list --consumed Status: Subscribed + valid Expires date No output / no consumed pools
repos --list-enabled BaseOS & AppStream listed Empty — nothing entitled
dnf repolist Returns repos with package counts Empty / network error / 403

The dnf repolist check is the ground truth. Everything above it can be green and this can still fail (proxy, DNS, certificate path, network ACL). If dnf doesn't return repos, the system is unpatchable — regardless of what subscription-manager says.

Phase 3 Common fixes
06 Repair by failure mode FIX ▶
Not registered subscription-manager register --org=<org> --activationkey=<key> No subscription attached subscription-manager attach --auto Expired or stale entitlements subscription-manager refresh Repos missing from list subscription-manager repos --enable=<repo>

After any fix, re-run the audit (Step 04). Don't skip the verification — a green status doesn't always mean dnf is happy, and the goal is patchability, not green dashboards.

07 Failure-state matrix REFERENCE ▶
Symptom Cause Fix
Not registered No identity register
status: Unknown No entitlement attach --auto
No consumed subs Nothing attached to pool attach --auto
No repos enabled No valid subscription / wrong arch repos --enable
dnf repolist empty Network / proxy / cert issue Check /etc/rhsm/rhsm.conf, proxy, firewall
SSL cert errors Stale CA / wrong system clock refresh · check date / NTP
Phase 4 Health check & pass criteria
08 One-liner health check VERIFY ▶
# Combines status + dnf check — useful for kickstart post-scripts and CI gates if subscription-manager status 2>/dev/null | grep -q "Current" && \ dnf repolist 2>/dev/null | grep -q "repo id"; then echo "LICENSE: OK" else echo "LICENSE: FAIL" fi

Use this in build pipelines. Both checks must pass — status alone is not enough. The grep "repo id" half catches network/proxy issues that subscription-manager can't see.

09 Minimal pass criteria CHECKLIST ▶
  • Registered — subscription-manager identity returns a UUID
  • Status — status reports Current
  • Subscription attached — list --consumed shows an active pool with a future Expires date
  • BaseOS & AppStream enabled — repos --list-enabled
  • Repos resolvable — dnf repolist returns one or more repos

All five must be true for the system to be considered patchable. Anything less and you're rolling the dice on the next dnf update.

⚠ Key Notes
  • Activation keys, not user/pass, in automation. Credentials in kickstart files or CI logs are an audit failure waiting to happen. Activation keys scope cleanly and rotate independently.
  • dnf repolist is the ground truth. Subscription-manager can be green while the system is still unpatchable due to proxy, DNS, or cert path issues. Always finish the audit at the dnf layer.
  • System clock matters. Subscription certificates are signed and time-bounded — a clock skew of more than a few minutes will cause SSL errors that look like network failures. Check date and NTP first when seeing certificate errors.
  • Repo names include the major version. rhel-8-for-x86_64-baseos-rpms ≠ rhel-9-for-x86_64-baseos-rpms. After in-place upgrades, repos must be re-enabled for the new major version.
  • refresh is cheap, run it after any subscription change. Updating pool membership in the Red Hat portal does not automatically update what's cached on the host — refresh pulls the new entitlements.
  • Don't unregister to "fix" things. Unregistering returns the entitlement to the pool but immediately disables repos and breaks patching. Try refresh, attach --auto, or repo re-enable first.
  • Proxies need /etc/rhsm/rhsm.conf. The http_proxy environment variable is not enough for subscription-manager — it reads its own config. Edit [server] proxy_hostname / proxy_port and restart rhsmcertd.
