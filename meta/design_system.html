01 Colour tokens

The palette is deliberately restrained: warm stone paper, one ochre accent, rust for caution, sage for verify, and a warm-dark code panel. Never hard-code a colour in a runbook; reference these variables. Never add a sixth accent family.

04 Notes and callouts

Tinted fill, matching 1px border, and a generated mono label. The variant class sets both the colour and the label text. Inline code lifts to --paper on a tinted callout so it stays legible.

Default .note, neutral background, NOTE label. For context and asides. Bold reads as ink.

.note-info: ochre. Rationale, "why this works", background the reader benefits from but isn't required to act on.

.note-caution: rust. A tradeoff or a place people get burned. Not necessarily a stop.

.note-critical: rust, harder. A genuine stop condition: do not proceed / do not reboot until this is true.

.note-verify: sage. The acceptance check. What "good" looks like after a step.

.note-success: sage. Confirmation the phase is complete.

05 Phase divider and step accordion

The .phase-label above is the section divider, a mono tag beside a Spectral heading, on a 2px ink rule. Below is the .step accordion. These are live: click a header to toggle it, hover a row to reveal its # deep-link anchor, and use the copy button on the code panel. All of that is wired by chrome.js, not by markup.

01 A step, open by default CRITICAL TARGET ▶

The header carries a numbered ochre badge, a title, one or more tags, an optional neutral run-on location chip, and a chevron. The body holds notes, code panels, and tables. Below is a code panel with the full syntax set, its header bar and copy button are built by chrome.js at load.

07 Warn box

The ochre-tinted .warn-box closes a runbook, a dash-led list of the things that most often bite. One per page, at the end.

⚠ Key Notes
  • Every component on this page is token-driven, toggle the theme and nothing hard-codes a colour.
  • Reach for the callout variant whose label matches intent: caution for a tradeoff, critical for a genuine stop, verify for an acceptance check.
  • Let chrome.js build code panels and accordions. Runbooks carry no copy script and no panel markup of their own.
